# BankGateway
Пет-проект для изучения микросервисной архитектуры и брокера сообщений Kafka. И изучения спецификации OpenAPI

# Основные задачи
Необходимо реализовать Банковский API-шлюз (API Gateway), который управляет запросами к множеству микросервисов (маршрутизация, агрегация ответов).
Ключевой функционал, который должен быть:
1) Протокольная трансляция: Преобразует данные из одного формата или протокола в другой (например, из REST API в SOAP, из XML в JSON)
2) Два микросервиса, помимо Банковского(BankApiGateway)
3) За маршрутизацию между микросервисами должна отвечать Kafka
4) Должен быть реализовать паттерн transactional outbox
5) Создать файл для описания контролера OpenApi
